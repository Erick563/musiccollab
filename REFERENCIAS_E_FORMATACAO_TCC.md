## 1. FORMATAÇÃO GERAL (ABNT NBR 14724:2011)

### 1.1 Configurações de Página
- **Papel**: A4 (21 cm × 29,7 cm)
- **Margens**: 
  - Superior: 3 cm
  - Inferior: 2 cm
  - Esquerda: 3 cm
  - Direita: 2 cm
- **Fonte**: Times New Roman ou Arial
- **Tamanho da fonte**: 12 pt (texto), 10 pt (citações longas, notas, legendas)
- **Espaçamento**: 1,5 entre linhas (texto), simples (citações longas, referências)
- **Parágrafo**: Recuo de 1,25 cm na primeira linha
- **Alinhamento**: Justificado

### 1.2 Estrutura do TCC

#### Elementos Pré-textuais (ordem obrigatória)
1. Capa
2. Folha de rosto
3. Folha de aprovação
4. Resumo (português)
5. Abstract (inglês)
6. Lista de figuras (se houver)
7. Lista de tabelas (se houver)
8. Lista de abreviaturas e siglas (se houver)
9. Sumário

#### Elementos Textuais
1. Introdução
2. Desenvolvimento (dividido em capítulos)
3. Conclusão

#### Elementos Pós-textuais
1. Referências (obrigatório)
2. Apêndices (se houver)
3. Anexos (se houver)

### 1.3 Numeração de Páginas
- Contar a partir da folha de rosto
- Numerar a partir da Introdução
- Números no canto superior direito, a 2 cm da borda

### 1.4 Títulos e Seções
- **Seção primária**: MAIÚSCULA, negrito, numerada (ex: 1 INTRODUÇÃO)
- **Seção secundária**: Maiúscula e minúscula, negrito, numerada (ex: 1.1 Contextualização)
- **Seção terciária**: Maiúscula e minúscula, negrito, numerada (ex: 1.1.1 Problema)
- **Seção quaternária**: Maiúscula e minúscula, sem negrito, numerada (ex: 1.1.1.1 Aspecto específico)

## 5. DICAS PARA REDAÇÃO ACADÊMICA

### 5.1 Linguagem

✅ **Use**:
- Linguagem impessoal (terceira pessoa)
- Voz passiva ou sujeito indeterminado
- Tempo verbal consistente (geralmente presente ou pretérito perfeito)
- Termos técnicos definidos previamente
- Parágrafos bem estruturados (introdução, desenvolvimento, conclusão)

❌ **Evite**:
- Primeira pessoa (eu, meu, nosso)
- Gírias e informalidades
- Abreviações não explicadas
- Repetições desnecessárias
- Parágrafos muito longos ou muito curtos

### 5.2 Exemplos de Redação

**❌ Inadequado**:
> Eu desenvolvi o sistema usando React porque eu acho que é a melhor ferramenta.

**✅ Adequado**:
> O sistema foi desenvolvido utilizando React.js devido à sua ampla adoção no mercado, ecossistema robusto e capacidade de criar interfaces de usuário reativas e componentizadas.

---

**❌ Inadequado**:
> O WebSocket é muito rápido e por isso escolhi ele pro projeto.

**✅ Adequado**:
> A tecnologia WebSocket foi selecionada para comunicação em tempo real devido à sua capacidade de estabelecer conexões bidirecionais full-duplex persistentes, reduzindo significativamente a latência em comparação com requisições HTTP tradicionais (SOCKET.IO, 2023).

---

### 5.3 Estrutura de Parágrafos

Cada parágrafo deve ter:
1. **Tópico frasal**: primeira frase que apresenta a ideia principal
2. **Desenvolvimento**: explicação, exemplos, argumentos
3. **Conclusão ou transição**: fechamento ou conexão com próximo parágrafo

**Exemplo**:

> A Web Audio API representa um avanço significativo no processamento de áudio em navegadores web. [tópico frasal] Essa API permite que desenvolvedores manipulem, analisem e sintetizem áudio diretamente no navegador sem necessidade de plugins, oferecendo recursos como roteamento de áudio, efeitos em tempo real e análise de frequências (MOZILLA, 2023). [desenvolvimento] Sua adoção no MusicCollab possibilitou a criação de um editor de áudio funcional e performático inteiramente baseado em tecnologias web. [conclusão]

## 6. FIGURAS E TABELAS

### 6.1 Figuras

Todas as figuras devem ter:
- Identificação na parte superior (Figura X - Título)
- Fonte na parte inferior (se não for autoral, Fonte: Autor, ano)
- Ser referenciadas no texto antes de aparecerem

**Exemplo**:

```
Figura 1 - Arquitetura do Sistema MusicCollab

[Imagem do diagrama]

Fonte: O autor (2024)
```

No texto:
> A arquitetura do sistema, ilustrada na Figura 1, é composta por três camadas principais...

### 6.2 Tabelas

Todas as tabelas devem ter:
- Identificação na parte superior (Tabela X - Título)
- Fonte na parte inferior
- Ser referenciadas no texto

**Exemplo**:

```
Tabela 1 - Comparação de Plataformas de Colaboração Musical

| Plataforma | Tempo Real | Cursores | Bloqueio |
|------------|-----------|----------|----------|
| Soundtrap  | Limitado  | Não      | Não      |
| MusicCollab| Sim       | Sim      | Sim      |

Fonte: O autor (2024)
```

### 6.3 Código-Fonte

Para trechos de código:
- Identificar como Quadro ou Código
- Usar fonte monoespaçada (Courier New)
- Incluir comentários explicativos
- Referenciar no texto

**Exemplo**:

```
Código 1 - Exemplo de Conexão WebSocket

// Conectar ao servidor WebSocket
const socket = io('http://localhost:3001', {
  auth: { token: authToken }
});

// Entrar na sala do projeto
socket.emit('join-project', projectId);

Fonte: O autor (2024)
```

## 8. CHECKLIST FINAL

Antes de entregar o TCC, verifique:

### Estrutura
- [ ] Todos os elementos pré-textuais estão presentes e na ordem correta
- [ ] Resumo e Abstract com máximo 500 palavras cada
- [ ] Sumário com paginação correta
- [ ] Capítulos seguem estrutura lógica
- [ ] Conclusão retoma objetivos e resultados

### Formatação
- [ ] Margens: 3-3-2-2 (superior, esquerda, inferior, direita)
- [ ] Fonte: Times New Roman ou Arial, tamanho 12
- [ ] Espaçamento: 1,5 entre linhas
- [ ] Numeração de páginas correta
- [ ] Títulos formatados conforme nível hierárquico

### Citações e Referências
- [ ] Todas as citações têm referência
- [ ] Todas as referências foram citadas no texto
- [ ] Referências em ordem alfabética
- [ ] Formatação ABNT correta em todas as referências
- [ ] URLs funcionam e data de acesso está presente

### Figuras e Tabelas
- [ ] Todas as figuras estão numeradas e têm título
- [ ] Todas as tabelas estão numeradas e têm título
- [ ] Todas as figuras/tabelas têm fonte
- [ ] Todas as figuras/tabelas são referenciadas no texto

### Conteúdo
- [ ] Objetivos estão claramente definidos
- [ ] Metodologia está bem descrita
- [ ] Resultados são apresentados de forma clara
- [ ] Conclusão responde aos objetivos
- [ ] Linguagem está formal e impessoal
- [ ] Texto está revisado (ortografia e gramática)

### Técnico (específico para TCC de sistemas)
- [ ] Diagramas estão claros e legíveis
- [ ] Código-fonte está formatado e comentado
- [ ] Decisões técnicas estão justificadas
- [ ] Tecnologias utilizadas estão descritas
- [ ] Arquitetura do sistema está documentada

## OBSERVAÇÕES FINAIS

1. **Consulte seu orientador**: Cada instituição pode ter requisitos específicos além das normas ABNT.

2. **Manual da sua instituição**: Verifique se há um manual de TCC específico da sua faculdade.

3. **Prazo para revisão**: Reserve pelo menos 2 semanas para revisão e ajustes finais.

4. **Revisão ortográfica**: Considere contratar um revisor profissional ou peça para colegas revisarem.

5. **Backup**: Mantenha múltiplas cópias do trabalho (nuvem, pen drive, e-mail).

6. **Versão PDF**: Gere uma versão PDF/A para arquivo final (preserva formatação).
